# Tutorial 12: ADK Artifacts - Save, Version, and Reuse Files

## Overview
Artifacts are ADK’s mechanism for storing durable file-like outputs outside regular conversational state. They are designed for binary or large payload workflows where session state alone is not enough.

With Artifacts, you can:
- Persist files generated by tools and agents
- Retrieve latest or specific versions of a file
- Organize data by artifact keys (filenames)
- Reuse outputs across multi-step workflows without regenerating every turn

This is useful for study guides, reports, checklists, drafts, media files, and other outputs that should survive beyond a single message response.

## Use Case: Artifact Notebook Agent
We implemented an **Artifact Notebook Agent** that behaves like a lightweight persistent workspace during an ADK Web UI session.

It supports two practical workflows:

1. **Versioned Note Management**
   - Create and save markdown notes (for example `rag-basics.md`)
   - Read latest or specific historical versions
   - Append new content while preserving older versions automatically
   - List saved note files to discover available artifacts

2. **Preference Memory in the Same Demo Context**
   - Save channel/style preferences into a dedicated artifact (`user:viewer_profile.md`)
   - Reload those preferences later in the same context

This mirrors common production patterns where agents must generate durable outputs and retrieve them reliably instead of repeating work each turn.

## Demo Prompts
Use these prompts in order:

1. `Create a concise study note on retrieval-augmented generation and save it as rag-basics.`
2. `List all saved notes.`
3. `Read rag-basics.md`
4. `Append 3 interview questions to rag-basics.md`
5. `Read rag-basics.md version 0`
6. `Read rag-basics.md` (latest version)
7. `Save my channel style preferences: practical tone, short intros, and code-first demos.`
8. `Load my saved channel style preferences.`

## Key Concepts
- `save_artifact(...)` writes a file and returns an incrementing version.
- Saving the same filename again creates a new version.
- `load_artifact(...)` can fetch latest or a specific version.
- `list_artifacts()` helps discover what exists.
- User-scoped artifacts can be created with filename namespacing such as `user:viewer_profile.md`.

## Notes
- This example focuses on text artifacts (`text/markdown`, `text/plain`).
- You can extend the same pattern for CSV, JSON, images, and PDFs by saving bytes in `inline_data`.
